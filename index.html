What your prompt should communicate

Instead of saying only “make me something like beat-battle.net for Logic Pro users,” you should describe the product as a competitive community platform for Logic Pro producers where users join beat battles, upload entries, vote on tracks, browse rankings, and feel like they’re inside a modern creative music environment.

01 / POSITIONING

Make the audience and purpose unmistakable

State that the site is specifically for Logic Pro beatmakers and producers. Mention that it should support battle discovery, challenge participation, submission deadlines, leaderboards, voting, and community credibility. This instantly moves the request from vague inspiration to product direction.

02 / VISUAL DIRECTION

Ask for a Logic-friendly music production aesthetic

Since Logic Pro has a sleek, pro-studio feel, the design should feel premium, rhythmic, modern, and software-inspired. Think dark interface, sharp typography, timeline motifs, mixer-like dividers, waveform visuals, and neon-lime or cyan accents rather than a generic startup layout.

03 / CORE PAGES

Specify the sections you actually want

Ask for a homepage with hero banner, current battle spotlight, submission CTA, featured producers, voting section, leaderboard, and FAQ. If you want more than a landing page, say you also need battle detail pages, producer profiles, upload flow, and community ranking views.

Shared loop battles that are fair

Every battle should provide the exact same official loop pack to every producer. Competitors download identical loops, build their beat in Logic Pro, then upload their final Logic project export and bounce so voters can compare skill, creativity, and mix choices on equal source material.

04 / LOOP PACK FLOW

Download the same loops, then build in Logic

Each challenge includes a locked starter pack (stems, one-shots, and BPM/key notes). Users click to download the official zip, produce inside Logic Pro, and return before deadline with a stereo bounce plus project package for transparent judging.

05 / HEAD-TO-HEAD COMPARE

Battle side-by-side with synchronized playback

The battle page should support A/B comparison between two producers using the same loop kit. Show waveform previews, quick switch controls, vote buttons, and upload verification badges so the audience knows both entries were created from the identical official source loops.
